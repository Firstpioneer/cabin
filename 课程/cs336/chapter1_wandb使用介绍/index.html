    Chapter1 wandb使用介绍

    W&B (Weights & Biases) 是一款专门为机器学习（ML）和深度学习（DL）开发者设计的 MLOps（机器学习运维）平台。 如果把编写代码比作开发软件，那么 W&B 就像是 “机器学习领域的 GitHub + TensorBoard + 代码版本管理” 的结合体。它旨在帮助研究人员和工程师记录实验、可视化结果、管理模型和数据集，并进行团队协作。 以下是 W&B 的主要功能和用途：

    1. 实验跟踪 (Experiment Tracking) —— 最核心的功能 在训练模型时，你需要尝试不同的超参数（如学习率、Batch Size）。W&B 可以自动记录这些信息。 指标可视化： 自动绘制损失函数（Loss）、准确率（Accuracy）等曲线。 超参记录： 记录每次运行（Run）所使用的配置。 系统监控： 实时监控 GPU/CPU 利用率、显存占用和网络状态。 代码对比： 记录实验对应的 Git commit，确保实验的可复现性。
    2. 超参数调优 (Sweeps) 手动调参非常耗时。W&B 的 Sweeps 功能可以自动化这个过程： 支持网格搜索（Grid Search）、随机搜索（Random Search）和贝叶斯优化（Bayesian Optimization）。 可以直观地看到哪些超参数对模型性能影响最大（通过平行坐标图等）。
    3. 数据集与模型版本管理 (Artifacts) 机器学习不仅仅是代码，数据和模型权重也需要版本控制。 数据集版本化： 记录模型训练时具体使用的是哪个版本的数据集。 模型仓库： 存储训练好的模型权重文件，方便随时调用或部署。 谱系追踪 (Lineage)： 清晰地查看“哪个数据集”训练出了“哪个模型”。
    4. 数据可视化与调试 (Tables) W&B Tables 允许你在浏览器中直接查看和分析数据： 预测结果对比： 在网页上直接对比不同模型对同一张图片或同一段文本的预测结果。 多媒体支持： 支持图像、视频、音频、3D 点云和 HTML 的交互式可视化。
    5. 协作与报告 (Reports) 实验总结： 可以像写博客或文档一样，将实验图表、结论和注释整合在一起。 团队共享： 团队成员可以查看彼此的实验进度，并在特定的实验运行下发表评论。
    6. 模型注册表 (Model Registry) 针对生产环境设计，管理模型的生命周期（如：开发中、预发布、已上线、已弃用），方便团队进行模型部署管理。
