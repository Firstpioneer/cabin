
Unicode是"编码标准"，为每个字符分配唯一码点；UTF-8是"编码格式"，负责将码点转换为字节序列。UTF-8的一大优势是：ASCII字符（0-127）在UTF-8中的编码与ASCII码完全一致，且只占1个字节。这种向后兼容的特性，让它比UTF-16、UTF-32等编码方式更为常用。

Token 并不固定是某一种形式，它取决于**分词器（Tokenizer）**的设置。
在英文中：
一个 Token 可能是一个完整的单词（如 apple）。
也可能是一个单词的一部分（如 learning 可能会被拆分为 learn 和 ing）。
甚至是一个标点符号。
规律： 1000 个英文 Token 大约等于 750 个单词。
在中文中：
一个 Token 可能是一个汉字（如 我）。
也可能是一个词语（如 机器人）。
现状： 不同的模型对中文的压缩率差异巨大。优秀的中文分词器会让一个 Token 包含更多的信息（比如一个词算一个 Token），从而提高压缩率。

为什么要用 Token，而不是直接用字符或单词？
这是为了在词表大小和语义理解之间寻找平衡：
如果以“字符”为单位： 粒度太细，模型要处理超长的序列，且单个字母/笔画没有意义。
如果以“单词”为单位： 词表会无限大（比如英文有各种时态变形，中文有无数词组），模型记不住这么多词。
使用 Token (子词拆分)： 比如 unhappy 拆成 un 和 happy。模型只要认识 un 代表“否定”，happy 代表“快乐”，就能推导出无数个带 un 前缀的词的意思。这大大节省了内存，提高了泛化能力。
